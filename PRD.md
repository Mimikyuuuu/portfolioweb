# 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品简介

本项目是一款将“个人作品集展示”与“2D 横屏探索小游戏”结合的互动式网页作品集。访客进入网站后，首先看到一个梦幻森林主题的横版地图，通过键盘控制原创角色左右移动，并与地图中的 4 个 NPC 交互，进入不同类型的作品展示页面。

MVP 阶段以“展示型作品集”为核心目标，优先跑通完整体验链路：游戏地图进入、NPC 交互、作品页浏览、返回地图并保留位置。项目暂不包含后端、数据库、登录、CMS 或移动端深度适配。

### 1.2 产品定位与核心价值

* **产品定位：** 兼具趣味性与专业度的体验式个人作品集。
* **核心价值：** 用可互动的游戏入口替代传统作品集首页，让访客在轻量探索中理解作者的内容能力、审美风格、前端实现能力与作品类型。
* **展示重点：** 游戏本身作为作品集体验外壳，作品页作为具体内容承载，两者共同体现作者的产品思维和视觉表达能力。

### 1.3 运行环境

* **平台：** PC 端 Web 浏览器，优先适配 Chrome、Edge、Safari。
* **操作媒介：** 键盘用于游戏地图控制，鼠标/触控板用于作品页浏览。
* **部署方式：** 静态网站部署到 Vercel。

## 2. MVP 范围

### 2.1 技术与产品边界

MVP 阶段采用 `React + TypeScript + Phaser 3`，以纯前端静态站点形式实现。所有作品内容通过本地配置文件、Markdown 和图片资源管理。

MVP 不包含：

* 后端服务、数据库、CMS 或管理后台。
* 用户登录、鉴权、JWT。
* 视频数据自动同步。
* 复杂平台跳跃、战斗、收集、任务系统。
* 移动端完整游戏适配。
* 个人信息、联系方式或简历入口。

### 2.2 MVP 内容规模

完整规划中的作品数量为：

* 游戏拆解：5 篇。
* 推送排版作品：3 篇。
* 社媒视频：2 个。
* 绘画作品：10+ 张。

MVP 阶段每个模块先接入 1 份真实素材，用于搭建完整体验：

| 模块 | MVP 内容 | 素材 |
| --- | --- | --- |
| 游戏拆解 | Pokopia 游戏拆解 | `public/articles/pokopia.md`、`public/articles/pokopia.jpeg` |
| 推送文章 | 九月 | `public/articles/September.png`、微信文章链接 |
| 视频作品 | 【旷野之心 x 蝴蝶舞蹈】暖暖，妈妈是野人！ | `public/videos/nikki.JPEG`、B 站链接、平台数据 |
| 绘画画廊 | 鹿野无料 | `public/gallery/luye.png` |

## 3. 核心游戏机制与地图架构

### 3.1 游戏控制机制

* **角色移动：** 键盘 `A` / `D` 或 `←` / `→` 控制角色左右移动。
* **不支持跳跃：** MVP 阶段只实现横向移动，不设计跳跃、平台、障碍或碰撞解谜。
* **NPC 交互：** 当角色进入 NPC 的触发范围时：
  * NPC 头顶显示 `!`。
  * 角色附近显示中文提示 `[E] 查看作品`。
  * 玩家按下 `E` 后进入对应作品页。

### 3.2 多场景切换与状态保存

项目采用“Phaser 游戏画布 + React DOM 页面”的混合模式。

* **进入作品页：** 触发交互后播放短暂淡出效果，暂停 Phaser 场景，隐藏游戏 Canvas，显示对应 React 作品页面。
* **退出作品页：** 每个作品页左上角固定显示 `← 返回地图` 按钮，并支持按 `ESC` 返回。
* **状态保持：** 返回地图后，玩家必须保持离开时的坐标位置，游戏不得重新加载或重置。
* **输入隔离：** 进入作品页后，游戏移动输入完全切断，页面切换为标准鼠标点击与滚动浏览模式。

### 3.3 地图与关卡设计

游戏采用单层连续横轴地图，主题为**梦幻森林**。画面要求简单、色彩舒适、温暖手绘风。

地图中设置 4 个 NPC 互动点：

```text
[出生起点]
   |
   |-- NPC 1：游戏拆解 -> 长文阅读页
   |
   |-- NPC 2：推送文章 -> 外链卡片页
   |
   |-- NPC 3：视频作品 -> 视频封面与数据页
   |
   |-- NPC 4：绘画画廊 -> 大图画廊页
   |
[终点]
```

### 3.4 角色与 NPC

* 玩家角色为原创形象，参考素材：
  * `figure.png`：正面形象参考。
  * `stand.png`：侧面站立参考。
  * `walk.png`：侧面行走动画参考。
* NPC 使用与玩家统一的温暖手绘风格，不需要设定具体身份。
* NPC 通过服装颜色区分不同作品模块。

## 4. 独立作品页功能需求

### 4.1 游戏拆解作品页

**目标：** 展示深度长文内容，强调可读性和专业排版。

**MVP 内容：**

* 标题：Pokopia 游戏拆解。
* Markdown：`public/articles/pokopia.md`。
* 封面：`public/articles/pokopia.jpeg`。

**页面布局：**

* 左侧为文章目录/文章列表。
* 右侧为 Markdown 正文阅读区。

**核心功能：**

* 点击左侧文章项后，右侧无刷新切换正文。
* Markdown 正文支持标题、段落、图片、引用块、加粗等基础富文本样式。
* MVP 只有 1 篇文章，但结构需支持后续扩展到 5 篇。

### 4.2 推送文章作品页

**目标：** 展示推送排版作品，并引导访客前往原生链接查看。

**MVP 内容：**

* 标题：九月。
* 日期：2024.10.11。
* 封面：`public/articles/September.png`。
* 链接：`https://mp.weixin.qq.com/s/oT5bF9JsCGuUpPRbsYVEgA`。

**页面布局：**

* 卡片式网格布局。
* 每张卡片包含封面图、标题和发布日期。

**核心功能：**

* 鼠标悬停时卡片轻微上移并增强边框或阴影反馈。
* 点击卡片后，在新标签页打开对应文章链接。
* MVP 只有 1 张卡片，但结构需支持后续扩展到 3 篇。

### 4.3 社媒视频作品页

**目标：** 用视频封面和数据面板展示视频创作与平台运营表现。

**MVP 内容：**

* 标题：【旷野之心 x 蝴蝶舞蹈】暖暖，妈妈是野人！
* 封面：`public/videos/nikki.JPEG`。
* 链接：`https://www.bilibili.com/video/BV1NcCdBGEmX/?share_source=copy_web&vd_source=fa8e02ed23b5d501820a9b9949ec639d`。
* 数据：
  * 小红书：686 播放、95 赞、16 收藏。
  * B 站：188 播放、22 赞、3 收藏。

**页面布局：**

* 上方为视频封面图与播放按钮。
* 下方为平台数据展示面板。

**核心功能：**

* 点击封面或播放按钮后，在新标签页打开视频链接。
* 数据面板展示不同平台的播放量、点赞数和收藏数。
* MVP 只有 1 个视频，但结构需支持后续扩展到 2 个。

### 4.4 绘画作品页

**目标：** 提供沉浸式大图画廊体验，突出绘画作品本身。

**MVP 内容：**

* 作品名：鹿野无料。
* 大图：`public/gallery/luye.png`。
* 简介：无。

**页面布局：**

* 中央显示当前大图。
* 旁边或下方展示作品名、创作时间、工具和简介。
* 底部保留缩略图预览条结构。
* 左右切换按钮在只有 1 张图时可禁用。

**核心功能：**

* 点击缩略图或切换按钮后切换主图。
* 主图切换使用淡入淡出效果。
* 支持键盘 `←` / `→` 切换图片。
* MVP 只有 1 张图，但结构需支持后续扩展到 10+ 张。

## 5. UI / UX 视觉与交互规范

### 5.1 游戏视觉

* 游戏场景采用梦幻森林主题。
* 美术风格为简单、舒适、温暖的手绘感。
* 玩家和 NPC 保持统一形象语言，NPC 通过服装颜色区分。
* 可使用轻量视差背景增强层次感，但 MVP 不强制复杂地图编辑。

### 5.2 作品页视觉

* 作品页采用现代、简洁、清爽的中文阅读与展示界面。
* 页面不展示个人信息、联系方式或额外营销内容。
* 内容区域优先保证可读性、图片展示质量和点击路径清晰。

### 5.3 通用交互

* 所有可点击的按钮、卡片、封面、缩略图都必须显示手型光标。
* 所有可点击元素应有轻微 Hover 动画反馈。
* `← 返回地图` 按钮在四个作品页中保持一致位置和样式。
* `ESC` 在作品页中始终返回地图。
* 作品页打开外部链接时使用新标签页，避免访客离开当前作品集站点。

## 6. 数据管理原则

### 6.1 静态数据驱动

MVP 阶段禁止将作品标题、图片路径、文章路径、视频链接和数据硬编码在组件中。所有内容通过本地配置文件统一管理。

建议使用：

* `src/data/projects.json`：管理作品数据与 NPC 配置。
* Markdown 文件：管理游戏拆解长文。
* 本地图片资源：管理封面、角色参考、绘画作品和视频封面。

### 6.2 后续可扩展性

数据结构需要为未来扩展做好准备：

* 游戏拆解可从 1 篇扩展到 5 篇。
* 推送文章可从 1 篇扩展到 3 篇。
* 视频作品可从 1 个扩展到 2 个。
* 绘画作品可从 1 张扩展到 10+ 张。
* 后续如升级后端，现有 JSON 字段应尽量可映射到 API 数据模型。

## 7. MVP 开发路线图

### 阶段一：基础游戏链路

* 建立 React 与 Phaser 的基础嵌入。
* 实现梦幻森林横版地图原型。
* 接入玩家站立与行走参考素材。
* 实现左右移动。
* 放置 4 个 NPC，并通过服装颜色区分。
* 实现靠近 NPC 显示 `[E] 查看作品`。
* 实现按 `E` 切换到对应作品页。
* 实现返回地图时恢复玩家原坐标。

### 阶段二：作品页与数据接入

* 设计 `projects.json` 数据结构。
* 接入 1 篇游戏拆解 Markdown。
* 接入 1 篇推送文章卡片。
* 接入 1 个视频封面和数据面板。
* 接入 1 张绘画作品。
* 四个作品页统一返回地图交互。

### 阶段三：视觉与动效打磨

* 优化梦幻森林色彩、背景层次与地图节奏。
* 优化角色/NPC 动画与方向切换。
* 加入场景淡入淡出效果。
* 优化卡片 Hover、画廊淡入淡出、按钮反馈。
* 检查键盘输入冲突和状态保留。

### 阶段四：Vercel 部署验收

* 验证静态资源路径在 Vercel 中可正常访问。
* 验证 Markdown、图片和外链正常加载。
* 验证刷新、返回地图、ESC、外链打开等核心路径。
* 验证 Chrome、Edge、Safari 的基础兼容性。
